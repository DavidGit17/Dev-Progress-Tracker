Build a mobile-first web application called “Dev Rebirth Tracker” — a daily discipline and productivity tracking system designed specifically for a developer.

CORE PURPOSE:
The app helps users plan their day, execute deep work sessions, track productive vs non-productive time, and visualize weekly performance.

FEATURE REQUIREMENTS:

1. DAILY TASK SYSTEM:

- Users can create tasks with:
  - Title
  - Category (Routine, Learning, Work, Interview Prep)
  - Start time
  - End time or duration
- Tasks should appear in a daily timeline view.

2. TIME BLOCKING + EXECUTION:

- Each task has a “Start Session” button.
- When started, a timer runs.
- Track actual time spent vs planned time.

3. NOTIFICATIONS:

- Trigger notifications at task start time.
- Notification sound should resemble an urgent “developer pager / on-call alert” tone (not soft reminder).
- Notifications should feel serious and action-driven.

4. DISTRACTION TRACKING:

- Allow user to log non-productive time manually.
- Include optional app usage tracking:
  - Store app name (e.g., YouTube, Instagram)
  - Duration of usage
- Classify this as “Non-Work Time”.

5. DAILY HISTORY:

- Store each day’s data persistently.
- Show:
  - Completed tasks
  - Missed tasks
  - Total productive vs non-productive time

6. WEEKLY ANALYTICS DASHBOARD:

- Provide visual insights:
  - Total productive vs wasted time
  - Category breakdown (coding, learning, prep, etc.)
- Include:
  - Bar chart (daily performance)
  - Pie chart (time distribution)
  - Line graph (progress trend)

7. UI / UX DESIGN:

- Theme: Black, White, Gray only (no colorful UI)
- Style inspired by Nothing UI:
  - Clean, modern, slightly futuristic
  - Use dotted / tech-style font where appropriate
- Use large buttons and touch-friendly layout
- Avoid overly minimal tiny icons — prioritize usability
- Use card-based layout with clear spacing

8. TECH STACK:

- React (with Vite)
- TypeScript
- Tailwind CSS
- Zustand or similar for state management
- Local storage or IndexedDB for persistence

9. ARCHITECTURE:

- Modular and scalable
- Designed for multiple future versions and feature expansions
- Clean separation between UI, logic, and data

10. FUTURE-READY:

- Code should be structured so new features can be added easily
- This is Version 1 of a product that will evolve with:
  - AI insights
  - Automation
  - Advanced tracking

OUTPUT:

- Generate UI screens, components, and basic logic
- Focus on usability, clarity, and performance
- Prioritize mobile responsiveness
